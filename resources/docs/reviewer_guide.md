# Reviewer Guide (Outline)

## Getting Access
- Roles that can review: reviewer, security_lead, org_admin, platform_admin
- Login/reset password

## Navigating the Queue
- Filters (status, urgent, category, date)
- Trash and restore

## Viewing a Case
- Activity History timeline (audit events)
- Files and previews (signed URLs)
- Contact info visibility (confidential vs anonymous)

## Messaging & Follow-Up
- Reply to reporter (sets first_response_at on first reply)
- Reporter follow-up portal link
- Attachments in thread (preview/download)

## Status Workflow
- Status options: Open → In review → Closed
- Adding status notes
- “Resolved by” (auto sets current reviewer if closing)
- Reopen clears resolver

## Urgent Handling
- Urgent badge, alerts sent to recipients
- How to re-trigger urgent alerts (via Org contacts / settings)

## Audit & Compliance
- Activity History entries for views, status changes, messages, attachments

## Tips
- Keep replies concise, link to dashboard
- Use notes to explain status changes
